CSS 预处理器

CSS预处理器是一种用于增强CSS编程体验的工具，它允许开发人员使用类似编程语言的语法来编写样式表，然后将其编译为标准的CSS文件。这些预处理器提供了一些强大的功能和语法，例如嵌套规则、变量、混合（Mixins）、函数等，可以显著提高CSS的可维护性和灵活性。

以下是一些常用的CSS预处理器：

  1. Sass (Syntactically Awesome Stylesheets): Sass是一种流行的CSS预处理器，它在CSS的基础上引入了变量、嵌套规则、混合、继承等功能，使得样式表更加易于维护和扩展。Sass还有两种语法格式：SCSS（Sassy CSS）和缩进格式（Indented Syntax），使开发人员可以根据自己的喜好选择不同的语法风格。
  2. Less: Less是另一种常见的CSS预处理器，它也提供了类似Sass的功能，包括变量、嵌套规则、混合等等。Less的语法比较简洁，与原生的CSS更加接近，因此对于熟悉CSS的开发人员来说，学习曲线可能会比较平缓。
  3. Stylus: Stylus是另一种CSS预处理器，它具有非常简洁的语法和很强的灵活性。使用Stylus可以轻松地编写复杂的样式表，并利用诸如变量、嵌套规则和混合等功能。

一、Sass

注意点

  1. sass变量后加!default表示默认变量值，当该sass文件被他人通过@import导入时，导入者可以修改这些变量值。即如果这个变量被声明赋值了，那就用它声明的值，否则就用这个默认值。

  2. sass提供了静默注释，以//开头，注释内容直到行末，其内容不会出现在生成的css文件中，/*...*/ 这种注释会出现在生成的css文件中。

  3. sass的混合器使用@mixin与@include

与Less的不同

  • Sass是在服务端处理的，以前是Ruby，现在是Dart-Sass或Node-Sass，而Less是需要引入less.js来处理Less代码输出css到浏览器，也可以在开发服务器将Less语法编译成css文件，输出css文件到生产包环境
  • 变量符不一样，Less是@，而Sass是$
  • Sass 在功能上更为强大，具有更多的内置函数，以及更加灵活的条件控制结构，例如if{}else{},for{}循环等等，而Less不支持
  • 工具库不同，Sass有工具库Compass，Less有UI组件库Bootstrap
