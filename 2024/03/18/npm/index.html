npm

npm（Node Package Manager）是 Node.js 平台上最常用的软件包管理器，用于安装、分享和管理 JavaScript 代码包。 它允许开发人员轻松地访问并安装来自 npm 注册表中的成千上万的软件包，从而加快了 JavaScript 应用程序的开发速度。

npm 由两个主要部分组成:

  • 用于发布和下载程序包的 CLI（命令行界面）工具
  • 托管 JavaScript 程序包的 在线存储库

CLI 常用命令

  1. npm init：初始化新的 npm 项目的命令。通过运行 npm init 命令，会自动生成 package.json 文件，其中包含了项目的基本设置和元数据信息。
  2. npm install：安装项目的所有依赖项。这个命令会读取 package.json 文件并安装指定的依赖，会自动生成 package-lock.json 文件。
  3. npm install xx：安装特定的软件包。可以使用这个命令来安装项目所需的特定依赖，例如 npm install lodash。
  4. npm update：更新项目的依赖项。这会根据 package.json 文件中指定的版本范围，安装最新的符合条件的依赖。
  5. npm uninstall ：卸载特定的软件包。可以使用这个命令来移除项目中不再需要的依赖。
  6. npm cache clean：清除整个 npm 缓存。当你执行这个命令时，将会清除所有已下载的包和缓存文件，并释放磁盘空间。在遇到诸如缓存文件错误、包无法下载或依赖项冲突等问题时，可能需要尝试 --force 强制清除缓存。
  7. npm run xx：运行自定义的脚本。可以使用这个命令来运行在 package.json 文件中定义的其他自定义脚本，例如 npm run build。
  8. npm publish：发布软件包到 npm 注册表。这个命令会将当前项目作为一个软件包发布到 npm 上，前提是你有相应的发布权限。
  9. npm version：更新软件包版本号。可以使用这个命令来更新软件包的版本号，例如 npm version patch 会将版本号的最后一位数字加一。
  10. npm audit：检查项目的依赖项是否存在已知的安全漏洞。这个命令可以帮助开发人员发现并解决项目中的潜在安全问题。
  11. npm config set registry https://registry.npmmirror.com：设置 npm 镜像源为淘宝镜像或其他适合你的镜像站点。
  12. npm config get registry：获取当前 npm 镜像源。
  13. npm config set proxy xx 或 npm config set https-proxy xx：为 npm 设置代理服务器。
  14. npm config get proxy 或 npm config get https-proxy：获取当前 npm 代理服务器。

安装常用标志 (flags)

  1. –save, -S：将依赖项保存到 dependencies 字段，即生产依赖。例如，npm install lodash --save。

  2. –save-dev, -D：将依赖项保存到 devDependencies 字段，即开发依赖。例如，npm install jest --save-dev。

  3. –global, -g：全局安装依赖项，使其在系统中的任何位置都可用。例如，npm install nodemon --global。

  4. –no-save：安装依赖项时不更新 package.json 文件中的依赖配置。

  5. –force：强制重新安装依赖项，忽略缓存。

  6. –production：仅安装生产依赖项，跳过安装开发依赖项。

  7. –legacy-peer-deps：在处理包的依赖关系时使用旧的、遗留的方式。它的作用是解决在使用新的依赖关系解析机制时可能遇到的兼容性问题。

    在较新的版本中，npm 引入了一种名为 “Peer Dependencies” 的依赖关系解析机制。这种机制允许开发者更细粒度地指定项目对其他包的依赖关系，以更好地管理包之间的关系。然而，一些旧的包可能不兼容这种新的机制，导致在安装这些包时出现一些问题。

    使用 --legacy-peer-deps 选项告诉 npm 在处理依赖关系时，采用旧的、更宽松的依赖解析策略。这样做可能会导致一些旧的包之间的依赖关系得到更宽松的解释，以兼容较旧的依赖管理方式。

  8. –ignore-scripts：忽略依赖包中的安装脚本。

  9. –prefer-offline：尝试从本地缓存安装依赖项，而不尝试从网络下载。

package.json 与 package-lock.json

package.json 文件是 Node.js 项目中的重要配置文件，通常位于项目的根目录下。它包含了项目的元数据和配置信息，以及项目所依赖的包和脚本命令。package-lock.json 是 npm 在安装包时自动生成的文件，它记录了项目依赖项的精确版本号信息和依赖树的结构。它的存在旨在确保在不同的环境中安装相同的依赖树结构，以保证项目的可重复性和稳定性。

以下是 package.json 文件中常见的字段和作用：

  1. name：项目的名称，通常采用小写字母和连字符的方式命名。
  2. version：项目的版本号，遵循语义化版本规范。
  3. description：项目的简要描述。
  4. main：指定项目的入口文件，如index.js。
  5. scripts：定义了一组可执行的脚本命令，例如启动、测试、构建等命令。
  6. dependencies：列出了项目运行时所依赖的包，即生产环境依赖。
  7. devDependencies：列出了在开发和构建时需要的依赖包，即开发环境依赖。
  8. keywords：关键词或标签，用于描述项目的特性。
  9. author：项目的作者信息。
  10. license：项目使用的许可证类型。
  11. repository：项目的代码仓库信息。
  12. engines：指定项目运行所需的 Node.js 版本范围。
  13. **…**：还可能包含其他自定义字段，用于存储项目的特定配置信息。

延伸：package.json 与 package-lock.json的关系

package.json 和 package-lock.json 两者之间有着紧密的关系，它们在管理和记录项目的依赖关系方面发挥着不同的作用。

  1. package.json：

    • package.json 文件是 Node.js 项目的核心配置文件，用于记录项目的元数据、依赖列表和脚本命令等信息。
    • 在 package.json 文件中，你可以定义项目的名称、版本、描述，以及依赖项、开发环境依赖项、脚本命令等内容。
    • 在 package.json 中列出的依赖项通常是使用语义化版本控制的范围形式。理解语义版本前面的符号非常重要。
      • ^：表示最新的次版本，例如， ^1.0.4 可能会安装主版本系列 1 的最新次版本 1.3.0。
      • 〜：表示最新的补丁程序版本，与 ^ 类似， 〜1.0.4 可能会安装次版本系列 1.0 的最新次版本1.0.7。
      • 所有这些确切的软件包版本都将记录在 package-lock.json 文件中。

  2. package-lock.json：

    • package-lock.json 文件在执行 npm install 命令时自动生成，记录了实际安装的每个依赖项的确切版本号、包的依赖树结构和相关的元数据信息。
    • 它的作用是确保在不同的环境中安装相同的依赖树结构，以保证项目的可重复性和稳定性，同时避免不经意的包版本更新。
    • package-lock.json 文件中的版本号是精确指定的，即使在 package.json 中定义的是范围形式的依赖版本，实际安装时也会根据 package-lock.json 中记录的确切版本进行安装。

因此，package.json 文件是项目依赖的描述和规划，而 package-lock.json 则是确保在不同的安装环境中使用相同的依赖进行安装，从而实现项目的稳定和可重复性。这两个文件共同作用于 Node.js 项目的依赖管理，保证了项目在不同环境中的一致性。
