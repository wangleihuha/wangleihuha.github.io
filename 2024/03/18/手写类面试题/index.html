JS手写类面试题

该类面试题的关键点在于首先要想清楚要实现的函数的输入、输出与作用。

一、手写map、filter、reduce

  1. map

    map 方法用于将数组中的每个元素按照指定规则映射/转换成另一个元素，返回转换后的新数组。

    关键点：

    • this指向很重要，看如何获取这个数组、及每一项的值
    • 数组如何修改不重要，如何修改都在callback里定义了，此处我们直接把callback想要的形参传给他就可以了~

  2. filter

    filter 方法用于筛选出数组中满足指定条件的元素，返回这些满足条件的元素构成的新数组。

  3. reduce

    reduce 方法的作用是对数组中的元素逐个执行回调函数，将回调函数的结果汇总为单个值，输出所有回调函数的结果与初始值的总和。

手写call、apply、bind

call、apply 和 bind 是 JavaScript 中用于改变函数执行上下文（this 指向）的方法。

实现这几个方法的关键点在于如何改变函数的this指向，我们首先回忆一下this指向相关知识点，this指向大致可以分为三种：

  • 第一种是函数调用模式，当一个函数不是一个对象的属性时，直接作为函数来调用时，this 指向全局对象。

  • 第二种是方法调用模式，如果一个函数作为一个对象的方法来调用时，this 指向这个对象。

  • 第三种是构造器调用模式，如果一个函数用 new 调用时，函数执行前会新创建一个对象，this 指向这个新创建的对象。

那这里我们可以借助第二种来实现，将这个函数作为传入的对象的属性，那它的this必然指向它啦~

三、手写防抖、节流

  1. 防抖

    防抖的基本思想是，在事件频繁触发的情况下，只有当事件停止触发一段时间后，才执行对应的处理函数。在这段时间内，如果事件再次触发，则重新计时。这样可以防止函数频繁执行。

    例子：假设监听用户输入搜索框的输入事件，当用户输入结束后1000毫秒才触发搜索请求。

  2. 节流

    节流的基本思想是，一段时间内只允许函数执行一次。在连续触发事件时，即使持续触发，也会在规定的时间间隔内执行一次函数。

    例子：假设监听页面滚动事件，每500毫秒执行一次处理函数。
